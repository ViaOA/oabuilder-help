Workflow Overview

"The design process is now the valuable part of the development process."

Creating applications with OA is a reiterative process, that supports Agile development. It's strength is in how productive it is to "grow" a project, modify it and rerun with very little programming. This allows more focus on the design process, that results in a much better product.

Create Model

  • Create a new model
  • Define model settings, name, package name, location to generate data, etc.
  • Select and define database settings(not required, can be added at any time)
  • Add new Objects
  • Define object properties, that define the data elements of the object.
  • Link objects together and define the link relationship properties
  • Define custom methods
  • Create filters
Next
  • Print reports
  • Generate database scripts, which includes an update script for changes maded that affect an existing database
  • Save model, stores as an XML file, with obx file name extension.
  • Generate database and project source code files.
  • Comple code
  • Run server application
  • Run client application
  • Access web application from browser
Changes
  • Make any chages to model
  • Update database with DDL script of changes
  • Regenerate application
  • Run application
