Link Editor

OABuilder creating "links" (references) between objects. These links for a relationship and rules for how model objects work together.

As with all of the editors in OABuilder, the default values are almost always what is needed, so there is no need to set each individual setting.

A link is created either by using the menu, right clicking on object and choose a link option, or by using the desktop menu's link command.




General tab, showing a regular reference link.

  • Type of reference, depending on how it was created.
  • Type - one or many. If many then checkboxes to define if: usually one, multi-select, large
  • Name
  • Upper name
  • Display
  • Cache size - allows limiting the number kept in cache.
  • default value
  • Processed - if this is created by code, and not UI.
  • Reset names - command to reset the default names.
  • Flag - to highlight the link in green.
  • Description



General tab, showing Hierarchy link.
This is created by right clicking an object and choosing the Hierarchy link. This will allow selecting property path through other links to a property that is defined in all of the links in the property path.
This allows a value to be set for all objects "below" it in the heirarchy.



General tab, showing Calculated link.
A calculated link uses code to get the reference object.

Properties (tab) is then used to determine what property paths are used to determined the value of the link.



General tab, showing Merger link. This is created by right clicking on an object, or using main menu.
A Merger link is used to create a relationship using a property path, so that all of the objects are combined into the collection.



General tab, showing GroupBy link. This is created by right clicking on an object, or using main menu.

  • The object selected is the GroupBy Object.
  • The first dialog is to pick the objects to add to the group.
  • A second dialog is to pick the property path between the objects.

For example, an Item can have a list of open Orders - property path "Orders:open()".
This is done by right clicking Item, choosing GroupBy link, then selecting property path for Object to select: "Orders:open()", and then a second dialog to pick property path between Order and Item ("Order.orderItems.item")



General II tab. FYI: most of the time, the default values are all that is needed.

  • Create method X.getY - flag to create a method.
  • Does X own Y - if the releationship is owned and treated as "one".
  • Is X required
  • Is X recursive - enabled if the link is to the same object, allows for creating a recursive relationship.
  • Is X an import match - if this link is used as unique value when importing data.
  • Sort X by - default sort property path for Many link.
  • Sequence property for X - for Many link, a property flaged as seq, to use to keep in order.
  • Unique property for X - for Many link, the property that is unique.



General III tab.

  • Ok to create a new X for Y.x
  • Ok to Auto Create a new X for X.Y - if One Link, then automatically create a new value.
  • Ok to add X for Y.X - allow assigning an existing object X to Y.X.
  • Default X from - property path to select the default value.
  • Hier - if property path is defined as a hierarchy.
  • Allow Change - if default value can be changed.
  • Auto Math X.Y using - property path used to auto create.
  • Select X from - property path to select from.
  • Equal proeprty path for X - property path that selected value must match.
  • Pick from link for creating X - the link that is required to create this link. Example: an Order has many OrderItems, added by selecting the Item for OrderItem.item
  • X Filter - filter to use for selecting from.


Properties tab. Used for Links that are calculated, to select the dependent property path(s) used to create the link's values.



Save/Delete tab. Rules for save and delete. Cascading is enabled for owned relationships.



From/To Code tabs.

  • X.get/setY methods - only needed to customize default code gen.
  • X.YCallback - callback method.



From/To Code tabs, User Interface.

  • X.Y User Interface - customize UI rendering.



Permissions tab.



DataSource tab.

  • Link Table Name - for M2M links.
  • From/To Fkey - DB foreign key names and constraints.
